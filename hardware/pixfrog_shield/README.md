# pixfrog shield — LED bus signal conditioning

KiCad 10 project for the signal-adaptation shield that plugs onto the
ESP32-P4 Module DEV-KIT's 2×20 header (J1). It closes the gap identified in
the TODO ("output buffers"): the devkit drives the LED bus straight from
3.3 V GPIOs, which is marginal for 12 V strips (WS2815) over real cable runs
and leaves the SoC exposed to field wiring.

![pixfrog shield](../../docs/img/pixfrog-shield.png)

## What it does

| Block | Parts | Role |
|---|---|---|
| Level shift | U1, U2 — 2× 74HCT245 (TSSOP-20) | TTL-threshold inputs driven cleanly by 3.3 V; outputs re-drive all 16 bus lines (8× DATA via U1, 8× CLOCK via U2) at 5 V — double the noise margin at the strip |
| Series termination, DATA | R1–R8 (249 Ω) + R9–R16 (39 Ω) behind SW1 | 249 Ω permanently in line (WS281x-style input resistor); closing the channel's DIP on SW1 parallels 39 Ω → ≈34 Ω for low-impedance drive of long lines ("33Ω/249Ω" on the silkscreen) |
| Series termination, CLOCK | R17–R24 (33 Ω) + R25–R32 (39 Ω) behind SW2 | 33 Ω base for clocked strips (APA102/SK9822/LPD8806); SW2 parallels 39 Ω → ≈18 Ω |
| Transient clamp | D1–D16 — 16× 5 V TVS (SMA) | One clamp to GND per output; a zap on the field wiring sacrifices a €0.10 diode, not the P4 |
| Outputs | J2–J9 — 8× JST-XH 3 pins | Per channel: 1 = DATA, 2 = CLOCK, 3 = GND (strip power stays on the strip PSU; signal ground must travel with the data pair) |
| Aux breakout | J13 — 1×9, 2 mm | 3V3, spare GPIOs 0/6/20/21/27/36/45, GND |

The channel→GPIO mapping mirrors `boards/esp32_p4_devkit.h` exactly
(CH1 DATA=GPIO2/CLK=GPIO3 … CH8 DATA=GPIO53/CLK=GPIO54).

## Production

`production/` is JLCPCB-ready, generated by the KiCad Fabrication Toolkit
(options in `fabrication-toolkit-options.json`):

- `PixFrog_Shield.zip` — gerbers + drill
- `bom.csv` / `positions.csv` — SMT assembly with LCSC part numbers
- `netlist.ipc`, `designators.csv`
