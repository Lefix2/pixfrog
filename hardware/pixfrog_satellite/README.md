# pixfrog satellite — LED signal repeater + power injection

KiCad 10 project for the remote node that sits at the **far end of a long LED
cable run**. The controller (or `pixfrog_shield`) sends one channel's two signal
lines down the cable; over distance the edges degrade and the strip voltage
sags. The satellite **re-squares the signal** with a Schmitt-trigger buffer and
**injects local strip power**, so the LEDs see clean data and full voltage
regardless of run length.

![pixfrog satellite](../../docs/img/pixfrog-sat.png)

It repeats **one pixfrog channel** = its two bus lines (`Data1`, `Data2`). What
they carry follows the protocol set on the controller:

- **WS281x (NRZ)** — `Data1` = DATA (`Data2` unused),
- **clocked (APA102 / SK9822 / LPD8806)** — `Data1` = DATA, `Data2` = CLOCK,
- **DMX512** — `Data1`/`Data2` = the differential pair.

Two voltage builds exist — **12 V** (WS2815-class) and **24 V** — chosen to match
the strip supply on `VCC`; the on-board LDO derives 5 V for the buffer from either.

## What it does

| Block | Parts | Role |
|---|---|---|
| Signal repeater | U1 — 74LVC2G17 (SC-74-6) | Dual non-inverting **Schmitt-trigger** buffer, powered at +5 V. Re-squares both lines (`Data1`, `Data2`) so the strip is driven by fresh 5 V-logic edges, not the degraded cable signal |
| Input series + clamp | R1, R2 (33 Ω) · D1, D2 (TVS) | 33 Ω in series into each buffer input; one 5 V bidirectional TVS per input line to GND |
| Output series, impedance select | R5, R6 (249 Ω) · R3, R4 (39 Ω) behind J4 | 249 Ω permanently in line on each output (WS281x-style). Fitting a jumper/switch on J4 (1‑2 and/or 3‑4) parallels 39 Ω → ≈34 Ω for low-impedance drive of long lines — same trick as the shield's SW1 |
| Output clamp | D3, D4 — TVS | One 5 V bidirectional TVS per output line to GND |
| Local 5 V | U2 — LD1117S50 (SOT-223) · C1 100 n, C2 10 µF (out) · C3 10 µF (in) | Linear 5 V from `VCC` (12 V or 24 V) — the logic rail for the buffer (`+5V` net, internal only) |
| Power pass-through / fan-out | J1 in → J2 + J5 | The strip supply (`VCC`) enters on J1 and is passed straight to the strip output (J2) and the J5 fan-out, GND common throughout |
| Mounting | H1, H2 — GND pads | Tie the board GND to the chassis/heatsink |

## Connectors

| Ref | Type | Pins | Use |
|---|---|---|---|
| J1 | 2-pin screw (Phoenix MKDS 5.08) | `VCC`, `GND` | Strip power **in** (12 V or 24 V per build) |
| J3 | 3-pin screw | `GND`, `Data1_in`, `Data2_in` | Signal **in** from the controller / shield channel |
| J2 | 4-pin screw | `VCC`, `Data1_out`, `Data2_out`, `GND` | **Out** to the LED strip — regenerated data + injected power |
| J5 | 2-pin JST-XH (`fan_out`) | `VCC`, `GND` | Power fan-out — chain to the strip's other end or a downstream node |
| J4 | 4-pin JST-XH | pre-/post-249 Ω taps | Optional impedance-select header (see table above); leave open for 249 Ω only |

## Signal + power flow (per line)

```
Data_in (J3) ─[TVS]─ 33Ω ─► 74LVC2G17 (@ +5V) ─► 249Ω ─[TVS]─ Data_out (J2)
                                                    │
                                              J4 ── 39Ω  (jumper ⇒ ≈34Ω)

VCC (J1, 12/24V) ─┬─► LD1117S50 ─► +5V (buffer)
                  └─► J2 (strip) + J5 (fan-out)
```
